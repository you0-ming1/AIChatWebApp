# 学习 Apple 的设计哲学

------

# UI 核心设计理念（非常重要）

# 1. UI 必须“安静”

界面不能：

- 太吵
- 太满
- 太亮
- 太炫

不能：

- 大面积高饱和颜色
- 全屏渐变
- 发光边框
- 赛博朋克灯条

真正高级 UI：

# 是克制的

------

# 2. 留白极其重要

Apple 风格最重要的是：

# 空间感

要求：

- 元素之间必须有呼吸感
- 不要贴边
- 不要挤
- 不要信息爆炸

错误示例：

```txt
按钮贴按钮
卡片贴卡片
文字挤成一团
```

正确目标：

# “舒服”

# “松弛”

# “高级”

------

# 3. 所有圆角必须统一

项目必须统一圆角系统。

推荐：

```txt
小元素：10px
普通卡片：16px
大容器：24px
弹窗：28px
```

禁止：

```txt
每个地方圆角都不一样
```

这样会失去高级感。

------

# 4. 动效必须柔和

动画核心：

# “轻”

# “短”

# “丝滑”

推荐：

```css
transition: all 0.2s ease;
```

或者：

```css
cubic-bezier(0.4, 0, 0.2, 1)
```

禁止：

- 夸张弹跳
- 疯狂旋转
- 大范围动画
- 重型动效

Apple 风格：

# 动画像空气流动

而不是：

# 网页蹦迪

------

# 5. 所有颜色必须柔和

禁止：

- 纯黑
- 纯白
- 高亮蓝
- 高亮红

推荐颜色：

# 深色模式背景

```txt
#0f1115
#151821
#1b1f27
#20232d
```

# 卡片颜色

```txt
rgba(255,255,255,0.04)
rgba(255,255,255,0.06)
```

# 边框颜色

```txt
rgba(255,255,255,0.08)
```

重点：

# 不要“死黑”

而是：

# 蓝灰色系

------

# 6. 毛玻璃（Glassmorphism）必须克制

允许：

- 微弱模糊
- 微弱透明
- 微弱高光

推荐：

```css
backdrop-filter: blur(20px);
```

但禁止：

- 全屏玻璃
- 超重模糊
- 彩色玻璃

否则：

# 会变成“Windows QQ 空间主题”

------

# 前端技术栈（UI部分）

# 核心技术

- Vue3
- TypeScript
- Vite
- TailwindCSS
- Element Plus（仅部分组件）
- SCSS

------

# UI 技术重点

# TailwindCSS

必须重点使用。

原因：

- AI 对 Tailwind 支持极强
- 开发效率极高
- 容易统一设计系统
- 容易维护

------

# 推荐 Tailwind 插件

推荐安装：

```txt
@tailwindcss/forms
@tailwindcss/typography
tailwind-scrollbar
tailwindcss-animate
```

------

# 图标库

推荐：

# Lucide Icons

原因：

- 极简
- Apple 风格
- 干净
- 线条统一

禁止：

- 花哨图标
- 厚重图标
- 彩色图标

------

# 字体系统（极其重要）

# 中文字体

推荐：

- MiSans
- HarmonyOS Sans
- 思源黑体

------

# 英文字体

推荐：

```txt
Inter
SF Pro Display（类似风格）
```

------

# 代码字体

推荐：

```txt
JetBrains Mono
```

------

# 字体原则

必须：

- 字重清晰
- 层级清晰
- 行高舒服
- 不要太密

推荐：

```css
line-height: 1.6;
```

------

# 页面布局（重要）

推荐布局：

# 三栏布局

```txt
左侧：
会话列表

中间：
聊天区域

右侧：
模型设置 / prompt / 参数
```

------

# 左侧边栏

风格：

- 半透明
- 微弱边框
- 柔和 hover
- 不要重阴影

会话 hover：

```css
background: rgba(255,255,255,0.06);
```

------

# 中间聊天区（重点）

聊天区域必须：

# 像“高级阅读器”

而不是：

# 微信聊天框

------

# AI 消息

特点：

- 更宽
- 更像文档
- 留白更多
- markdown 漂亮

------

# 用户消息

特点：

- 稍微高亮
- 更小
- 更聚焦

------

# Markdown 渲染（极其重要）

AI 产品：

# 核心视觉体验来自 markdown

必须重点优化：

- code block
- 表格
- 引用
- 标题
- 列表

------

# Code Block 风格

代码块必须：

- 有圆角
- 有背景层次
- 有复制按钮
- 有语言标签

禁止：

```txt
纯黑代码框
```

------

# 输入框设计（重点）

输入框：

必须：

- 大圆角
- 半透明
- 有呼吸感
- 支持自动高度

参考：

- ChatGPT Mac
- Claude

推荐：

```css
border-radius: 24px;
```

------

# 按钮风格

按钮：

必须：

- 圆润
- 微阴影
- 微 hover
- 不要太亮

推荐：

```css
transform: translateY(-1px);
```

轻微悬浮感。

------

# 滚动条（重要）

必须美化滚动条。

推荐：

- 超细
- 半透明
- hover 才明显

禁止：

```txt
Windows 默认滚动条
```

会瞬间破坏高级感。

------

# 阴影系统

Apple 风格：

# 阴影很轻

推荐：

```css
box-shadow:
0 4px 20px rgba(0,0,0,0.12);
```

禁止：

- 重黑阴影
- 浮夸阴影

------

# 动效系统（极其重要）

所有动画：

必须：

# “像液体”

而不是：

# “像PPT”

------

# 动画原则

推荐：

- 150ms
- 200ms
- ease
- 柔和渐变

禁止：

- 超长动画
- 夸张 bounce
- 疯狂 easing

------

# 流式输出动画（重点）

AI 输出时：

必须：

- 逐字显示
- cursor 闪烁
- 微弱 fade

效果：

```txt
正在思考 ▌
```

增强“AI 正在工作”的感觉。

------

# 页面整体气质

最终目标：

# “像 Apple 官方 AI 产品”

用户打开后：

第一感觉：

# “高级”

# “舒服”

# “专业”

# “安静”

而不是：

# “炫”

# “炸”

# “酷”

高级感来源于：

- 克制
- 统一
- 留白
- 细节
- 动效
- 字体
- 层级

------

# AI 生成 UI 的规则（非常重要）

以后 AI 生成前端代码时：

必须：

# 1. 不允许默认 Element 样式

必须自定义。

------

# 2. 不允许“后台管理系统风”

禁止：

- 蓝色按钮海
- 大表格
- 大边框
- 工具网站风格

------

# 3. 所有 UI 必须解释设计原因

AI 必须解释：

- 为什么这样布局
- 为什么这样配色
- 为什么这样动效
- 为什么这样留白

------

# 4. 所有 CSS 必须详细注释

包括：

- 为什么使用 blur
- 为什么使用 rgba
- 为什么使用 transition

------

# 5. 所有组件必须具有统一设计语言

包括：

- 圆角
- 阴影
- 动效
- 字体
- 间距

必须统一。

------
